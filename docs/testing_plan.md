# Testing Plan

## Objective
The purpose of testing is to ensure that the AI Disease Prediction System works correctly and provides reliable disease predictions.

## Testing Types

### 1. Dataset Testing
- Verify Training.csv and Testing.csv files load correctly.
- Check for missing or incorrect values.
- Confirm symptom and disease columns are properly formatted.

### 2. Model Testing
- Test the machine learning model using sample symptom inputs.
- Verify that predictions are generated correctly.
- Measure model performance using evaluation metrics.

### 3. Interface Testing
- Check whether users can enter symptoms.
- Verify prediction results are displayed properly.
- Ensure the interface is easy to use.

### 4. Error Testing
- Test empty symptom input.
- Test invalid inputs.
- Ensure the system handles errors properly.

## Expected Outcome
The system should accept symptoms, process them, and provide accurate disease predictions.
